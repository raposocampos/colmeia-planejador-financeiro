# Checklist de QA

## Engenharia

- [ ] TypeScript estrito sem erros.
- [ ] ESLint e Prettier aprovados.
- [ ] Unitários e componentes aprovados.
- [ ] E2E desktop e mobile aprovados.
- [ ] Builds Sites e GitHub Pages aprovados.
- [ ] Sem segredos ou .env versionado.

## Produto

- [ ] Onboarding completo, pular e reiniciar.
- [ ] CRUD de conta, cartão, transação, orçamento e meta.
- [ ] Dashboard reage e persistência sobrevive à recarga.
- [ ] Filtros e relatórios coerentes.
- [ ] JSON/CSV exportam; importações mostram prévia.
- [ ] Dados demo são removíveis.

## Marca e acessibilidade

- [ ] Nome oficial em toda a interface.
- [ ] Paleta e tipografia coerentes com o brandbook.
- [ ] Hexágonos controlados e sem aparência genérica de fintech.
- [ ] Contraste, foco, labels, teclado e reduced motion verificados.
- [ ] Gráficos possuem alternativa textual.

## Publicação

- [ ] GitHub Actions aprovado.
- [ ] GitHub Pages carrega JS, CSS, manifest e service worker.
- [ ] Sites privado publicado e aberto.
- [ ] Console publicado sem erro relevante.
